## 🔄 ПОШАГОВАЯ ИНСТРУКЦИЯ

### 1. Добавить точки взлёта

**Способ 1: Через модальное окно**
1. Нажать "Добавить точку взлёта"
2. Ввести координаты или нажать "Моё местоположение"
3. Ввести название (например, "База А")
4. Нажать "Сохранить"

**Способ 2: Программно**
```javascript
MultiRouteModule.addTakeoffPoint({
    lat: 55.75,
    lon: 37.61,
    name: 'База А',
    priority: 1,
    voltage: 25.4,
    capacity: 39000,
    antenna: 60
});
```

### 2. Загрузить маршруты

**Способ 1: Загрузка KML**
1. Нажать "Загрузить KML"
2. Выбрать файл на компьютере
3. Маршрут автоматически добавится в список

**Способ 2: Выбор из сохранённых**
1. Кликнуть на сохранённый маршрут в списке
2. Маршрут добавится в мульти-маршрут

**Способ 3: Создать новый**
1. Ввести координаты точек A и B
2. Нажать "Создать маршрут"
3. Маршрут сохранится и добавится в список

### 3. Нажать "Анализировать все маршруты"

**Что происходит:**
1. ✅ Проверка: есть ли маршруты и точки взлёта
2. ✅ Основной анализ: 1-й маршрут → `stepData.segmentAnalysis`
3. ✅ Мульти-маршрут анализ: все маршруты + все точки
4. ✅ Оптимизация распределения по базам
5. ✅ Переход к шагу 2 (Пилот)

## ⚠️ ВАЖНЫЕ ЗАМЕЧАНИЯ

### 1. Первый маршрут — основной

Для совместимости с существующей логикой:
- **1-й загруженный маршрут** становится `stepData.route`
- **Основной анализ** выполняется для 1-го маршрута
- **Шаг 2 (Пилот)** работает с 1-м маршрутом

### 2. Мульти-маршрут — дополнительный анализ

- **Все маршруты** анализируются дополнительно
- **Оптимизация** учитывает все точки взлёта
- **Результат** доступен в дашборде (вкладка "Мульти-маршрут")

### 3. Требования для анализа

```
✅ Хотя бы 1 точка взлёта
✅ Хотя бы 1 маршрут
✅ Выбрана дата анализа
✅ Получены метеоданные
```
