# Калькулятор прямой видимости антенн

**Файл:** `line-of-sight-test.html`

## Возможности

✅ **Расчет радиогоризонта** по формуле `d ≈ 4.12 × √h` (с учетом рефракции k=4/3)

✅ **Расчет максимальной дальности** связи между двумя антеннами

✅ **Учет рельефа местности** — данные SRTM через Open-Elevation API

✅ **Зона Френеля** — расчет при указании частоты

✅ **Визуализация**:
- Интерактивная карта OpenLayers
- График профиля трассы
- Маркер препятствия при отсутствии видимости

## Как использовать

1. Откройте `line-of-sight-test.html` в браузере
2. Введите координаты передающей антенны (широта, долгота)
3. Укажите высоту антенны над землей (м)
4. Введите координаты приемной точки
5. Укажите высоту приемной антенны (м)
6. При необходимости укажите частоту (МГц) для расчета зоны Френеля
7. Нажмите **"Рассчитать видимость"**

## Результаты

| Параметр | Описание |
|----------|----------|
| Расстояние | Расстояние между точками (км) |
| Радиогоризонт TX | Дальность горизонта передающей антенны (км) |
| Радиогоризонт RX | Дальность горизонта приемной антенны (км) |
| Макс. дальность | Сумма радиогоризонтов (км) |
| Зона Френеля | Радиус первой зоны Френеля (м) |
| Препятствие | Высота препятствия над линией видимости (м) |
| Статус | Видимость есть / Нет видимости |

## Формулы

### Радиогоризонт
```
d = 4.12 × √h
```
где `h` — высота антенны над землей (м), `d` — расстояние до горизонта (км)

### Максимальная дальность связи
```
d_max = 4.12 × (√h₁ + √h₂)
```

### Зона Френеля
```
r = 0.6 × √(λ × d₁ × d₂ / (d₁ + d₂))
```
где `λ = 300/f` — длина волны (м), `f` — частота (МГц)

## Алгоритм проверки видимости

1. Получаем профиль высот вдоль трассы (50 точек)
2. Строим линию прямой видимости между антеннами
3. Для каждой точки профиля:
   - Вычисляем высоту линии видимости
   - Вычитаем радиус зоны Френеля (если указана частота)
   - Сравниваем с высотой рельефа
4. Если рельеф выше линии видимости — видимости нет

## API

Используется **Open-Elevation API** для получения высот:
- `https://api.open-elevation.com/api/v1/lookup`
- Данные основаны на SRTM (разрешение ~30м)

**⚠️ Важно:** API может быть недоступен. В этом случае используется режим эмуляции рельефа:
- Для Москвы: ~150м (равнина)
- Для Сочи/Кавказа: ~800-1500м (горы)
- Для Урала: ~400-800м (среднегорье)
- Для Крыма: ~300-800м (горы)
- Для Санкт-Петербурга: ~20м (низменность)
- Для Сибири: ~300-600м (плато)

Для реальных проектов рекомендуется использовать локальные DEM-данные (SRTM, ASTER).

## Ограничения

⚠️ **Open-Elevation API** может быть недоступен - используется эмуляция рельефа

⚠️ Точность эмуляции ограничена - для демонстрационных целей

⚠️ Не учитываются здания, деревья и другие объекты на местности

⚠️ Для продакшена используйте реальные DEM-данные (SRTM 30м или лучше)
