### 3. ⚠️ MET Norway API (CORS проблема)

**Проблема:** CORS ошибка при запросе к api.met.no

```
Access to fetch at 'https://api.met.no/...' from origin 'http://127.0.0.1:5508' 
has been blocked by CORS policy: No 'Access-Control-Allow-Origin' header
```

**Причина:** MET No API не поддерживает CORS запросы напрямую из браузера.

**Временное решение:**
- Автоматическая загрузка отключена
- MET No карточка показывает заглушку с данными Open-Meteo
- Требуется прокси-сервер для работы

**Как включить (требуется бэкенд):**
```javascript
// wizard.js: applyPilotCorrectionAndShowAnalysis()
// Раскомментировать строку:
this.loadMetNoForecast(); // Было закомментировано
```

**Для полноценной интеграции нужно:**
1. Поднять прокси-сервер (Node.js/Python)
2. Настроить CORS заголовки
3. Обновить `metno.js` для работы через прокси

### 1. CORS MET No API

**Статус:** 🔴 Критично  
**Решение:** Требуется прокси-сервер

**Варианты решения:**

**A. CORS-прокси (быстро, нестабильно):**
```javascript
const proxy = 'https://cors-anywhere.herokuapp.com/';
const url = proxy + 'https://api.met.no/...';
```

**B. Свой прокси-сервер (надёжно):**
```javascript
// Node.js Express прокси
app.get('/api/metno', async (req, res) => {
    const {lat, lon, msl} = req.query;
    const response = await fetch(
        `https://api.met.no/...?lat=${lat}&lon=${lon}&msl=${msl}`,
        {headers: {'User-Agent': 'MIRA/0.1.5.0 (kkav45@ya.ru)'}}
    );
    const data = await response.json();
    res.setHeader('Access-Control-Allow-Origin', '*');
    res.json(data);
});
```
