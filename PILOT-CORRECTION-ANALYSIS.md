## 🔍 Общий процесс коррекции

### Порядок расчёта:

```
1. Пользователь вводит данные с датчиков на шаге 3
   ↓
2. Нажимает "Применить коррекцию"
   ↓
3. WeatherModule.applyPilotCorrection() рассчитывает поправки
   ↓
4. Поправки применяются ко ВСЕМ сегментам маршрута
   ↓
5. Пересчитываются ВСЕ временные часы прогноза (24-48 часов)
   ↓
6. Вкладки шага 3 показывают обновлённый анализ
```

## 📐 Математическая модель коррекции

### 1. Расчёт коэффициентов коррекции

```javascript
applyPilotCorrection(analyzedData, pilotData) {
    const corrections = {
        // Коэффициент смещения ветра (отношение факт/прогноз)
        windBias: pilotData.windSpeed / analyzedData.hourly[0].wind10m,
        
        // Смещение температуры (разница факт-прогноз)
        tempOffset: pilotData.temp - analyzedData.hourly[0].temp2m,
        
        // Переопределение видимости (если туман)
        visibilityOverride: pilotData.fog ? 0.5 : null
    };
}
```

### 2. Применение коррекции ко времени

```javascript
corrected.hourly = corrected.hourly.map((hour, i) => {
    const weight = Math.exp(-i / 24); // Экспоненциальное затухание
    
    return {
        ...hour,
        wind10m: hour.wind10m * (1 + (corrections.windBias - 1) * weight),
        temp2m: hour.temp2m + (corrections.tempOffset * weight),
        visibility: corrections.visibilityOverride !== null
            ? corrections.visibilityOverride
            : hour.visibility
    };
});
```

## 📈 Весовой коэффициент (weight)

### Формула затухания:
```
weight = e^(-i / 24)
```

где `i` - номер часа от текущего момента (0, 1, 2, ...)

### Таблица затухания:

| Время от текущего | i (час) | weight | Доля коррекции |
|-------------------|---------|--------|----------------|
| Сейчас            | 0       | 1.000  | 100%           |
| +1 час            | 1       | 0.959  | 96%            |
| +3 часа           | 3       | 0.882  | 88%            |
| +6 часов          | 6       | 0.779  | 78%            |
| +12 часов         | 12      | 0.606  | 61%            |
| +24 часа          | 24      | 0.368  | 37%            |
| +36 часов         | 36      | 0.223  | 22%            |
| +48 часов         | 48      | 0.135  | 14%            |

### График затухания:

```
Коррекция (%)

    │ \

    │   \
```

## 🎯 Пример расчёта

### Входные данные:

**Прогноз Open-Meteo (на текущий час):**
- Ветер: 8 м/с
- Температура: +5°C
- Видимость: 5 км

**Фактические данные пилота:**
- Ветер: 11 м/с
- Температура: +3°C
- Туман: нет

### Расчёт коэффициентов:

```javascript
windBias = 11 / 8 = 1.375  // +37.5% к ветру
tempOffset = 3 - 5 = -2°C  // -2°C к температуре
visibilityOverride = null  // без изменений
```

### Применение к прогнозу:

| Время | weight | Ветер (прогноз → скорр.) | Темп. (прогноз → скорр.) |
|-------|--------|--------------------------|--------------------------|
| 0 ч   | 1.000  | 8 → **11.0 м/с**         | +5 → **+3.0°C**          |
| 1 ч   | 0.959  | 9 → **12.0 м/с**         | +4 → **+2.0°C**          |
| 3 ч   | 0.882  | 10 → **12.6 м/с**        | +3 → **+1.2°C**          |
| 6 ч   | 0.779  | 12 → **14.1 м/с**        | +2 → **+0.4°C**          |
| 12 ч  | 0.606  | 10 → **12.3 м/с**        | +1 → **-0.2°C**          |
| 24 ч  | 0.368  | 8 → **9.5 м/с**          | 0 → **-0.7°C**           |
| 48 ч  | 0.135  | 7 → **7.5 м/с**          | -1 → **-1.3°C**          |

## 📊 Применение к сегментам маршрута

### Порядок обработки:

```javascript
// wizard.js: applyPilotCorrectionAndShowAnalysis()
for (let i = 0; i < this.stepData.segmentAnalysis.length; i++) {
    const original = this.stepData.segmentAnalysis[i].analyzed;
    const corrected = WeatherModule.applyPilotCorrection(original, pilotData);
    
    correctedSegments.push({
        ...this.stepData.segmentAnalysis[i],
        analyzed: corrected,
        corrected: true
    });
}
```

### Что происходит:

1. **Берётся каждый сегмент** маршрута (обычно 6-10 сегментов по 10 км)
2. **Для каждого сегмента** применяется ОДНА и ТА ЖЕ коррекция пилота
3. **Коэффициенты коррекции одинаковы** для всех сегментов
4. **Затухание по времени** работает независимо для каждого сегмента

### Важное замечание:

❌ **Проблема**: Коррекция применяется по координатам точки взлета ко ВСЕМ сегментам

```
Точка взлета (пилот) → Сегмент 1 (10 км) → Сегмент 2 (20 км) → ... → Сегмент 6 (60 км)
     ↓                      ↓                   ↓                        ↓
  Одинаковая коррекция для ВСЕХ сегментов!
```

✅ **Реальность**: Метеоусловия могут отличаться на расстоянии 60 км

## 🏆 Приоритет данных (доверие)

### Иерархия доверия:

| Уровень | Источник | Доверие | Применение |
|---------|----------|---------|------------|
| **1**   | Пилот (факт) | 100% | Текущий момент (i=0) |
| **2**   | Пилот + Open-Meteo | 100% → 37% | 0-24 часа |
| **3**   | Open-Meteo (прогноз) | 63% → 86% | 24-48 часов |

### Формула доверия:

```
Доверие к факту = weight * 100%
Доверие к прогнозу = (1 - weight) * 100%
```

| Время | weight | Доверие факту | Доверие прогнозу |
|-------|--------|---------------|------------------|
| 0 ч   | 1.000  | 100%          | 0%               |
| 6 ч   | 0.779  | 78%           | 22%              |
| 12 ч  | 0.606  | 61%           | 39%              |
| 24 ч  | 0.368  | 37%           | 63%              |
| 48 ч  | 0.135  | 14%           | 86%              |

## ⚠️ Ограничения текущей модели

### 1. **Одна точка коррекции для всех сегментов**

**Проблема**: Данные пилота применяются ко всем сегментам маршрута, даже если они за 50-100 км.

**Пример**:
```
Пилот: Москва (ветер 5 м/с)
Сегмент 6: 60 км севернее (реальный ветер 12 м/с)
Результат: Заниженный риск для сегмента 6
```

### 2. **Экспоненциальное затухание**

**Проблема**: Через 24 часа коррекция составляет только 37%.

**Сценарий**:
- Пилот ввёл данные в 10:00
- Полёт планируется на 14:00 (через 4 часа)
- weight = 0.846 → коррекция 85%
- ✅ Хорошо для краткосрочных полётов

- Полёт планируется на следующий день в 10:00 (через 24 часа)
- weight = 0.368 → коррекция 37%
- ⚠️ Большая часть коррекции потеряна

### 3. **Только 3 параметра коррекции**

**Корректируются**:
- ✅ Ветер (wind10m)
- ✅ Температура (temp2m)
- ✅ Видимость (при тумане)

**НЕ корректируются**:
- ❌ Осадки
- ❌ Влажность
- ❌ Давление
- ❌ Облачность
- ❌ Турбулентность
- ❌ Обледенение

## 📋 Выводы

### ✅ Преимущества:

1. **Мгновенная коррекция** текущего прогноза
2. **Экспоненциальное затухание** - физически обосновано
3. **Применение ко всем сегментам** - учёт фактора времени
4. **Автоматический пересчёт** рекомендаций и графиков

### ⚠️ Недостатки:

1. **Одна точка замера** для всего маршрута
2. **Короткий горизонт** эффективной коррекции (12-24 часа)
3. **Ограниченное количество** корректируемых параметров

### 🎯 Рекомендации:

1. **Вводить данные перед взлётом** (не раньше чем за 1-2 часа)
2. **Использовать для локальных полётов** (до 30 км от точки взлета)
3. **Не полагаться на коррекцию** для прогноза >24 часов
4. **Проверять сегменты вручную** если маршрут >50 км
