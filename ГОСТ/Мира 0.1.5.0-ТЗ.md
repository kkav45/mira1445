# 3. НАЗНАЧЕНИЕ РАЗРАБОТКИ

## 3.1. Функциональное назначение

Система предназначена для решения следующих задач:

1. **Получение метеорологических данных:**
   - Прогноз погоды от Open-Meteo API
   - Данные по 4 высотам (250, 350, 450, 550 м)
   - Актуальные данные с геолокации пилота

2. **Анализ метеоусловий:**
   - Анализ ветра (наземного и высотного)
   - Анализ осадков и видимости
   - Анализ риска обледенения
   - Анализ турбулентности
   - Анализ температуры и влажности

3. **Планирование маршрутов:**
   - Построение маршрутов по нескольким точкам
   - Сегментация маршрута (участки по 10 км)
   - Пространственное распределение рисков
   - Вертикальный профиль рисков

4. **Коррекция прогноза:**
   - Учёт фактических данных с позиции пилота
   - Расчёт коэффициентов коррекции
   - Применение поправок к прогнозу

5. **Формирование отчётности:**
   - Экспорт отчётов в PDF
   - Интерактивные графики и диаграммы
   - Рекомендации по полёту

## 3.2. Область применения

Система применяется для:
- Предполётного планирования и анализа рисков
- Мониторинга метеоусловий вдоль маршрута
- Обучения пилотов БВС основам метеорологии
- Документирования метеоусловий полёта

**Система не предназначена для:**
- Использования в качестве единственного источника метеоинформации
- Реального времени во время полёта
- Сертифицированных полётов гражданской авиации

## 4.2. Требования к функциям (опциям)

### 4.2.1. Функция получения метеоданных

**Назначение:** Получение прогноза от Open-Meteo API

**Входные данные:**
- Широта (lat), градусы
- Долгота (lon), градусы
- Дата и время начала прогноза
- Количество дней прогноза (1-7)

**Выходные данные:**
- Температура (°C) на высотах 2м, 250м, 350м, 450м, 550м
- Влажность относительная (%)
- Точка росы (°C)
- Ветер 10м (м/с), направление (градусы)
- Ветер 500м (м/с)
- Порывы ветра (м/с)
- Осадки (мм/ч)
- Видимость (м)
- Облачность (%)
- Нижняя граница облаков (м)

**Требования к точности:**
- Температура: ±0,1°C
- Ветер: ±0,1 м/с
- Влажность: ±1%
- Видимость: ±100 м

### 4.2.2. Функция анализа рисков

**Назначение:** Расчёт Risk Score для каждого параметра

**Параметры анализа:**

| Параметр | Порог LOW | Порог MEDIUM | Порог HIGH | Вес |
|----------|-----------|--------------|------------|-----|
| Ветер 10м | ≤10 м/с | 10-15 м/с | >15 м/с | 1,5 |
| Ветер 500м | ≤15 м/с | 15-20 м/с | >20 м/с | 1,2 |
| Осадки | ≤1,4 мм/ч | 1,4-5 мм/ч | >5 мм/ч | 1,3 |
| Видимость | ≥5 км | 2-5 км | <2 км | 1,4 |
| Обледенение | LOW | MEDIUM | HIGH | 1,5 |
| Турбулентность | LOW | MEDIUM | HIGH | 1,3 |

**Выходные данные:**
- Risk Score (0-8 баллов)
- Уровень риска (LOW/MEDIUM/HIGH)
- Статус полёта (РАЗРЕШЁН/ОГРАНИЧЕН/ЗАПРЕЩЁН)

### 4.2.3. Функция построения маршрута

**Назначение:** Создание и управление маршрутами

**Возможности:**
- Добавление точек маршрута (минимум 2, максимум 50)
- Автоматическая сегментация (участки по 10 км)
- Расчёт дистанции между точками
- Отображение на карте (OpenLayers)
- Экспорт/импорт в KML

**Входные данные:**
- Координаты точек (широта, долгота)
- Название маршрута
- Базовый аэродром (опционально)

**Выходные данные:**
- Список сегментов с координатами
- Дистанция каждого сегмента
- Общий километраж маршрута

### 4.2.4. Функция коррекции прогноза

**Назначение:** Уточнение прогноза по фактическим данным

**Входные данные (фактические):**
- Ветер (м/с), направление (градусы)
- Температура (°C)
- Влажность (%)
- Видимость (км)
- Нижняя граница облаков (м)
- Признаки: туман, осадки, снег

**Алгоритм коррекции:**
```
1. Расчёт коэффициента ветра: windBias = windActual / windForecast
2. Расчёт поправки температуры: tempOffset = tempActual - tempForecast
3. Расчёт коэффициента влажности: humidityBias = humidityActual / humidityForecast
4. Применение с затуханием к высотам:
   correctedValue = forecastValue × bias + offset × weight
   где weight = exp(-altitude / 1000)
5. Принудительная коррекция видимости и облачности
```

**Выходные данные:**
- Скорректированный прогноз по высотам
- Рекомендации по максимальной высоте полёта
- Обновлённый Risk Score

### 4.2.5. Функция экспорта отчёта

**Назначение:** Формирование PDF-отчёта

**Состав отчёта:**
1. Титульная страница (название, дата, маршрут)
2. Статус полёта и рекомендации
3. Метеопараметры (таблица)
4. Информация о маршруте

6. Графики (временные ряды)
7. Вертикальный профиль
8. Наблюдения пилота (если есть)

**Технические требования:**

- Кодировка: UTF-8
- Шрифты: с поддержкой кириллицы
- Размер: ≤5 МБ
