# 1. ВВЕДЕНИЕ

## 1.1. Назначение документа

Настоящее руководство системного программиста (далее — РСП) предназначено для системных администраторов и специалистов, осуществляющих развёртывание, настройку, техническое обслуживание и поддержку системы MIRA v0.1.5.0 (далее — Система).

Документ разработан в соответствии с требованиями ГОСТ 19.506-79.

## 1.2. Область применения

РСП содержит:
- Назначение и условия применения Системы
- Порядок развёртывания и настройки
- Проверку работоспособности
- Техническое обслуживание
- Устранение неисправностей

## 1.3. Краткое описание Системы

MIRA — веб-приложение (SPA) для метеорологического обеспечения полётов БВС.

**Тип приложения:** Single Page Application (SPA)
**Язык реализации:** JavaScript (ES6+)
**Хранение данных:** localStorage браузера
**Внешние зависимости:** Open-Meteo API, OpenLayers, Plotly.js

### 3.1.4. Вариант 4: Произвольный веб-сервер

**Требования к серверу:**
- Поддержка статических файлов
- MIME-типы: .html, .css, .js, .json
- HTTPS (рекомендуется)

**Пример конфигурации Nginx:**
```nginx
server {
    listen 80;
    server_name mira.example.com;
    
    root /var/www/mira;
    index desktop.html;
    
    location / {
        try_files $uri $uri/ /desktop.html;
    }
    
    # MIME-типы
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires max;
        log_not_found off;
    }
    
    # Кэширование
    location ~* \.(js|css)$ {
        add_header Cache-Control "public, max-age=31536000";
    }
}
```

**Пример конфигурации Apache (.htaccess):**
```apache
RewriteEngine On
RewriteBase /

# Перенаправление на desktop.html
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^.*$ /desktop.html [L,QSA]

# MIME-типы
AddType application/javascript js
AddType text/css css

# Кэширование
<IfModule mod_expires.c>
    ExpiresActive On
    ExpiresByType application/javascript "access plus 1 year"
    ExpiresByType text/css "access plus 1 year"
</IfModule>
```

| Направление | Протокол | Порт | Домен | Действие |

| Исходящее | HTTPS | 443 | api.open-meteo.com | Разрешить |
| Исходящее | HTTPS | 443 | cdn.jsdelivr.net | Разрешить |
| Исходящее | HTTPS | 443 | cdnjs.cloudflare.com | Разрешить |
| Исходящее | HTTPS | 443 | openstreetmap.org | Разрешить |

**Пример для Windows Firewall (PowerShell):**
```powershell
# Разрешение исходящих соединений
New-NetFirewallRule -DisplayName "MIRA Open-Meteo" `
    -Direction Outbound `
    -RemoteAddress api.open-meteo.com `
    -Protocol TCP `
    -RemotePort 443 `
    -Action Allow

New-NetFirewallRule -DisplayName "MIRA CDN" `
    -Direction Outbound `
    -RemoteAddress cdn.jsdelivr.net,cdnjs.cloudflare.com `
    -Protocol TCP `
    -RemotePort 443 `
    -Action Allow
```

# 6. УСТРАНЕНИЕ НЕИСПРАВНОСТЕЙ

## 6.1. Диагностика проблем

### 6.1.1. Приложение не загружается

**Симптомы:**
- Белый экран
- Ошибки в консоли

**Действия:**
```
1. Открыть консоль браузера (F12)
2. Проверить наличие ошибок
3. Проверить загрузку скриптов (вкладка Network)
4. Проверить доступность CDN

Возможные причины:
  - Блокировка CDN брандмауэром
  - Устаревший браузер
  - Отключён JavaScript
```

**Решение:**
```javascript
// Проверка доступности CDN
fetch('https://cdn.jsdelivr.net/npm/ol@v9.2.4/dist/ol.js')
    .then(r => console.log('CDN доступен:', r.ok))
    .catch(e => console.error('CDN недоступен:', e));
```

### 6.1.2. Не получается прогноз

**Симптомы:**
- Ошибка «Failed to fetch»
- Пустые данные

**Действия:**
```
1. Проверить подключение к интернету
2. Проверить доступность API
3. Проверить консоль на ошибки CORS
4. Проверить координаты
```

**Решение:**
```javascript
// Проверка API
fetch('https://api.open-meteo.com/v1/forecast?latitude=55.75&longitude=37.61&current_weather=true')
    .then(r => r.json())

    .catch(e => console.error('API недоступен:', e));
```

2. Проверить настройки приватности

### 6.1.4. Не сохраняется маршрут

**Симптомы:**
- После перезагрузки маршрут исчезает

**Действия:**
```
1. Проверить доступность localStorage
2. Проверить квоту хранилища
3. Проверить режим инкогнито
4. Очистить устаревший кэш
```

**Решение:**
```javascript
// Проверка localStorage
try {
    localStorage.setItem('test', '1');
    localStorage.removeItem('test');
    console.log('localStorage доступен');
} catch (e) {
    console.error('localStorage недоступен:', e);
}

// Проверка квоты
if (navigator.storage) {
    navigator.storage.estimate().then(e => {
        console.log('Квота:', e.quota / 1e6, 'МБ');
        console.log('Использовано:', e.usage / 1e6, 'МБ');
    });
}
```

## 6.2. Типовые ошибки

### Ошибка 1: CORS

```
Сообщение: Access to fetch at '...' from origin '...' has been blocked by CORS policy

Причина: Запрос с локального файла

Решение:
  - Использовать локальный веб-сервер
  - Или загрузить на GitHub Pages
```

### Ошибка 2: QuotaExceededError

```
Сообщение: QuotaExceededError: Failed to execute 'setItem' on 'Storage'

Причина: Превышена квота localStorage (5-10 МБ)

Решение:
  - Очистить кэш: StorageModule.clearExpiredCache()
  - Удалить старые маршруты
  - Экспортировать и удалить данные
```

### Ошибка 3: TypeError: Cannot read property

```
Сообщение: TypeError: Cannot read property 'hourly' of undefined

Причина: Данные не загружены

Решение:
  - Проверить получение прогноза
  - Добавить проверку на undefined
  - Обработать ошибку API
```
