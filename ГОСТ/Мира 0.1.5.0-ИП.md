# 2. ОБЛАСТЬ ПРИМЕНЕНИЯ

## 2.1. Назначение Системы

Система MIRA предназначена для автоматизации метеорологического обеспечения полётов беспилотных воздушных судов (БВС), включая:

- Получение метеорологических прогнозов
- Анализ рисков для полёта
- Планирование маршрутов с учётом погоды
- Коррекцию прогноза по фактическим данным
- Формирование отчётной документации

## 2.2. Категории пользователей

| Категория | Описание | Примеры использования |
|-----------|----------|----------------------|
| Пилоты БВС | Операторы, управляющие БВС | Предполётный анализ, оценка рисков |
| Планировщики | Специалисты по планированию полётов | Построение маршрутов, выбор времени |
| Инструкторы | Преподаватели курсов пилотирования | Обучение метеорологии, разбор случаев |
| Исследователи | Аналитики метеоусловий | Сбор статистики, анализ тенденций |

## 2.3. Сценарии использования

### Сценарий 1: Предполётный анализ

**Описание:** Пилот планирует полёт и нуждается в оценке метеоусловий.

**Действия:**
1. Определение местоположения полёта
2. Получение прогноза погоды
3. Анализ рисков
4. Принятие решения о полёте

**Результат:** Статус полёта (разрешён/ограничен/запрещён) и рекомендации.

### Сценарий 2: Планирование маршрута

**Описание:** Необходимо построить оптимальный маршрут с учётом метеоусловий.

**Действия:**
1. Построение маршрута по точкам
2. Сегментация маршрута
3. Анализ рисков для каждого сегмента
4. Корректировка маршрута при необходимости

**Результат:** Маршрут с оценкой рисков на всём протяжении.

### Сценарий 3: Коррекция по фактическим данным

**Описание:** Пилот находится на месте и имеет фактические данные о погоде.

**Действия:**
1. Ввод фактических данных
2. Применение коррекции к прогнозу
3. Получение уточнённых рекомендаций

**Результат:** Скорректированный прогноз с повышенной точностью.

### Сценарий 4: Документирование

**Описание:** Требуется сохранить отчёт о метеоусловиях для отчётности или анализа.

**Действия:**
1. Формирование отчёта
2. Экспорт в PDF
3. Сохранение или печать

**Результат:** PDF-отчёт со всеми данными и рекомендациями.

# 3. ФУНКЦИОНАЛЬНЫЕ ВОЗМОЖНОСТИ

## 3.1. Обзор функций

Система предоставляет следующие основные функции:

| № | Функция | Краткое описание |
|---|---------|------------------|
| 1 | Геолокация | Определение текущего местоположения |
| 2 | Построение маршрута | Создание маршрутов по нескольким точкам |
| 3 | Получение прогноза | Загрузка данных от Open-Meteo API |
| 4 | Анализ рисков | Расчёт Risk Score по параметрам |
| 5 | Коррекция прогноза | Уточнение по фактическим данным |
| 6 | Визуализация | Графики, карты, диаграммы |
| 7 | Экспорт PDF | Формирование отчётов |
| 8 | Сохранение данных | Локальное хранение в браузере |

## 3.2. Подробное описание функций

### 3.2.1. Геолокация

**Назначение:** Определение координат пользователя для получения прогноза.

**Возможности:**
- Автоматическое определение через браузер
- Ручной ввод координат
- Выбор точки на карте

**Точность:**
- GPS (мобильные): 5-20 м
- Wi-Fi: 50-100 м
- IP: 1-5 км

**Ограничения:**
- Требуется разрешение браузера
- Не работает в режиме инкогнито
- Зависит от доступности GPS

### 3.2.2. Построение маршрута

**Назначение:** Создание маршрута полёта по контрольным точкам.

**Возможности:**
- Добавление до 50 точек
- Автоматическая сегментация (~10 км на сегмент)
- Редактирование (перемещение, удаление, добавление точек)
- Сохранение и загрузка маршрутов
- Импорт/экспорт в формате KML

**Параметры маршрута:**
- Минимальное количество точек: 2
- Максимальное количество точек: 50
- Автоматический расчёт дистанции
- Отображение азимута между точками

**Ограничения:**
- Максимальная длина маршрута не ограничена
- Точки должны быть в допустимом диапазоне координат

### 3.2.3. Получение прогноза

**Назначение:** Загрузка метеорологических данных от Open-Meteo.

**Параметры прогноза:**

| Параметр | Единицы | Диапазон |
|----------|---------|----------|
| Температура | °C | -50...+50 |
| Влажность | % | 0...100 |
| Ветер 10м | м/с | 0...50 |
| Ветер 500м | м/с | 0...50 |
| Осадки | мм/ч | 0...100 |
| Видимость | м | 0...50000 |
| Облачность | % | 0...100 |

**Временные характеристики:**
- Горизонт прогноза: до 7 дней
- Дискретность: 1 час
- Время получения: 2-5 секунд

**Источники данных:**
- Open-Meteo API (бесплатно, без ключа)
- Модели: GFS, ECMWF, ICON

**Ограничения:**
- Требуется подключение к интернету
- Прогноз является моделью, а не фактическими данными
- Точность снижается с увеличением горизонта прогноза

### 3.2.4. Анализ рисков

**Назначение:** Оценка рисков для безопасного полёта.

**Анализируемые параметры:**

| Параметр | Порог LOW | Порог MEDIUM | Порог HIGH |
|----------|-----------|--------------|------------|
| Ветер 10м | ≤10 м/с | 10-15 м/с | >15 м/с |
| Ветер 500м | ≤15 м/с | 15-20 м/с | >20 м/с |
| Осадки | ≤1,4 мм/ч | 1,4-5 мм/ч | >5 мм/ч |
| Видимость | ≥5 км | 2-5 км | <2 км |
| Обледенение | LOW | MEDIUM | HIGH |
| Турбулентность | LOW | MEDIUM | HIGH |

**Расчёт Risk Score:**
- Минимум: 0 баллов
- Максимум: 8 баллов
- Каждый параметр вносит 0-2 балла

**Уровни риска:**
- LOW (0-2): Низкий риск, полёт разрешён
- MEDIUM (3-5): Средний риск, полёт ограничен
- HIGH (6-8): Высокий риск, полёт запрещён

**Результат анализа:**
- Статус полёта
- Risk Score
- Детализация по параметрам
- Рекомендации

### 3.2.5. Коррекция прогноза

**Назначение:** Уточнение прогноза по фактическим данным.

**Входные данные:**
- Ветер (м/с, направление)
- Температура (°C)
- Влажность (%)
- Видимость (км)
- Нижняя граница облаков (м)
- Флаги: туман, осадки, снег

**Алгоритм коррекции:**
1. Расчёт коэффициентов (windBias, tempOffset, humidityBias)
2. Применение к прогнозу с затуханием по высоте
3. Принудительная коррекция видимости и облачности
4. Пересчёт рисков

**Результат:**
- Скорректированный прогноз
- Обновлённый Risk Score
- Рекомендации по максимальной высоте

**Эффективность:**
- Повышение точности на 15-25%
- Особенно эффективно при местных особенностях погоды

### 3.2.6. Визуализация

**Назначение:** Представление данных в наглядном виде.

**Типы визуализаций:**

| Тип | Описание | Использование |
|-----|----------|---------------|
| Карта | OpenLayers с маркерами и маршрутами | Отображение местоположения, маршрутов |
| Временные ряды | Графики параметров по времени | Анализ изменений погоды |
| Вертикальный профиль | Параметры по высотам | Оценка условий на разных высотах |
| Тепловая карта | Цветовое кодирование рисков | Быстрая оценка рисков |
| Таблицы | Детальные данные по сегментам | Точные значения |

**Интерактивность:**
- Масштабирование карты
- Приближение графиков
- Подсказки при наведении
- Переключение слоёв

### 3.2.7. Экспорт PDF

**Назначение:** Формирование печатных отчётов.

**Состав отчёта:**
1. Титульная страница
2. Статус полёта и рекомендации
3. Метеопараметры (таблица)
4. Информация о маршруте
5. Сегменты с рисками
6. Графики и профили
7. Наблюдения пилота (если есть)

**Технические характеристики:**
- Формат: PDF A4
- Кодировка: UTF-8 (поддержка кириллицы)
- Размер: обычно 1-3 МБ
- Страниц: 2-6 в зависимости от состава

**Возможности:**
- Выбор разделов для включения
- Печать напрямую
- Сохранение на диск
- Отправка по email

### 3.2.8. Сохранение данных

**Назначение:** Локальное хранение данных между сессиями.

**Хранимые данные:**
- Маршруты (список и координаты)
- Последний анализ
- Данные пилота
- Наблюдения (сидя на земле, в полёте)
- Настройки приложения

**Технология:** localStorage браузера

**Объём:**
- Доступно: ~5 МБ
- Типичное использование: 0,5-1 МБ

**Срок хранения:**
- Маршруты: бессрочно
- Кэш погоды: 1 час
- Наблюдения: бессрочно

# 5. УСЛОВИЯ ПРИМЕНЕНИЯ

## 5.1. Общие условия

Система может применяться при соблюдении следующих условий:

1. **Наличие подключения к интернету**
   - Для получения прогноза от Open-Meteo
   - Для загрузки библиотек (OpenLayers, Plotly.js)

2. **Поддерживаемый браузер**
   - Chrome, Firefox, Edge ≥90
   - Safari ≥14

3. **Разрешения браузера**
   - Геолокация (опционально, но рекомендуется)
   - localStorage (обязательно)
   - Cookies (обязательно)

## 5.2. Условия для конкретных задач

### 5.2.1. Предполётный анализ

**Требуется:**
- Подключение к интернету
- Координаты места полёта (геолокация или вручную)

**Не требуется:**
- Построение маршрута
- Коррекция по фактическим данным

### 5.2.2. Планирование маршрута

**Требуется:**
- Подключение к интернету
- Построение маршрута (минимум 2 точки)

**Не требуется:**
- Геолокация (можно вручную указать точки)
- Коррекция

### 5.2.3. Коррекция прогноза

**Требуется:**
- Выполненный анализ
- Фактические данные с места полёта
- Подключение к интернету

**Не требуется:**
- Построение маршрута

### 5.2.4. Экспорт отчёта

**Требуется:**
- Выполненный анализ
- Браузер с поддержкой pdfmake

**Не требуется:**
- Подключение к интернету (после получения данных)

## 5.3. Не рекомендуемые условия

**Не рекомендуется использовать Систему:**

| Условие | Причина |
|---------|---------|
| Без проверки из других источников | Прогноз может быть неточным |
| Для реального времени во время полёта | Система для планирования |
| При отсутствии интернета | Данные не загрузятся |
| В устаревших браузерах | Возможны ошибки |
| В режиме инкогнито | Данные не сохранятся |
| Для сертифицированных полётов | Не является сертифицированным средством |

# 6. РЕЖИМЫ ПРИМЕНЕНИЯ

## 6.1. Режимы работы

### 6.1.1. Режим «Быстрый анализ»

**Назначение:** Быстрая оценка условий перед полётом.

**Действия:**
1. Геолокация (3 сек)
2. Анализ (5 сек)
3. Просмотр статуса

**Время:** ~10 секунд

**Результат:** Статус полёта и краткие рекомендации.

### 6.1.2. Режим «Планирование»

**Назначение:** Детальное планирование полёта.

**Действия:**
1. Построение маршрута
2. Анализ по сегментам
3. Изучение графиков
4. Выбор оптимального времени
5. Экспорт отчёта

**Время:** 5-15 минут

**Результат:** Полный отчёт с маршрутом и рекомендациями.

### 6.1.3. Режим «Коррекция»

**Назначение:** Уточнение прогноза по фактическим данным.

**Действия:**
1. Ввод фактических данных
2. Применение коррекции
3. Просмотр обновлённых рекомендаций

**Время:** 2-3 минуты

**Результат:** Скорректированный прогноз с повышенной точностью.

### 6.1.4. Режим «Обучение»

**Назначение:** Изучение метеорологии и принципов анализа.

**Действия:**
1. Построение различных маршрутов
2. Сравнение прогнозов для разных мест
3. Изучение графиков и профилей
4. Анализ влияния параметров на риски

**Время:** Не ограничено

**Результат:** Понимание метеорологических факторов.

## 6.2. Выбор режима

**Рекомендации по выбору режима:**

| Ситуация | Рекомендуемый режим |
|----------|---------------------|
| Вы на месте, скоро взлетать | Быстрый анализ |
| Планируете полёт на завтра | Планирование |
| Есть данные с датчиков | Коррекция |
| Изучаете метеорологию | Обучение |

## Приложение Б. Типовые конфигурации применения

### Конфигурация 1: Мобильная

**Устройство:** Смартфон/планшет
**Браузер:** Chrome/Safari мобильный
**Сценарий:** Быстрый анализ на месте
**Особенности:**
- Использование геолокации GPS
- Сенсорное управление
- Компактный интерфейс

### Конфигурация 2: Десктоп

**Устройство:** ПК/ноутбук
**Браузер:** Chrome/Firefox/Edge
**Сценарий:** Детальное планирование
**Особенности:**
- Большой экран
- Точное управление мышью
- Расширенные возможности визуализации

### Конфигурация 3: Смешанная

**Устройство:** ПК + мобильное
**Сценарий:** Планирование на ПК, коррекция на месте
**Особенности:**
- Маршрут сохраняется в localStorage
- Доступ с разных устройств
- Экспорт/импорт KML для синхронизации

```
□ Открыть MIRA

□ Изучить рекомендации
□ При необходимости применить коррекцию
□ Экспортировать отчёт (опционально)

```

## Приложение Г. Глоссарий

| Термин | Определение |
|--------|-------------|
| БВС | Беспилотное воздушное судно |
