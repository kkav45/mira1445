# 3. БАЗОВЫЕ ОПЕРАЦИИ

## 3.1. Определение местоположения

### Способ 1: Автоматическая геолокация

**Когда использовать:** Вы находитесь на месте планируемого полёта.

```
Шаг 1. Нажмите кнопку 📍 Моё местоположение

Шаг 2. Браузер запросит разрешение на доступ
       к геолокации:
       ┌─────────────────────────────────┐
       │ Mira запрашивает доступ к       │
       │ вашему местоположению           │
       │                                 │
       │ [❌ Заблокировать] [✅ Разрешить]│
       └─────────────────────────────────┘
       
       Нажмите «Разрешить»

Шаг 3. Дождитесь определения координат
       (3-10 секунд)

Шаг 4. Карта центрируется на вашей позиции
       Координаты отобразятся в панели

Результат:
  ✅ Координаты определены
  ✅ Прогноз загружен автоматически
  ✅ Можно выполнять анализ
```

**Точность определения:**

| Метод | Точность | Время |
|-------|----------|-------|
| GPS (смартфон/планшет) | 5-20 м | 5-10 сек |
| Wi-Fi | 50-100 м | 3-5 сек |
| Сотовые вышки | 500-2000 м | 2-3 сек |
| IP-адрес | 1-5 км | 1-2 сек |

### Способ 2: Выбор точки на карте

**Когда использовать:** Планируете полёт в другом месте.

```
Шаг 1. Наведите курсор на нужное место на карте

Шаг 2. Кликните левой кнопкой мыши
       Появится маркер с координатами

Шаг 3. При необходимости перетащите маркер
       в более точное место

Шаг 4. Координаты зафиксируются
       Можно выполнять анализ

Результат:
  ✅ Координаты установлены
  ✅ Можно получать прогноз
```

### Способ 3: Ручной ввод координат

**Когда использовать:** Знаете точные координаты.

```
Шаг 1. Нажмите на поле ввода координат
       (обычно в верхней части панели)

Шаг 2. Введите координаты в формате:
       55.7558, 37.6173
       
       где:
       55.7558 — широта (градусы)
       37.6173 — долгота (градусы)

Шаг 3. Нажмите Enter или кнопку «Перейти»

Шаг 4. Карта переместится к указанной точке

Примеры координат:
  Москва:        55.7558, 37.6173
  Санкт-Петербург: 59.9343, 30.3351
  Казань:        55.7961, 49.1064
```

**Форматы координат:**

| Формат | Пример | Поддержка |
|--------|--------|-----------|
| Десятичные градусы | 55.7558, 37.6173 | ✅ |
| Градусы и минуты | 55°45.35', 37°37.04' | ⚠️ |
| Градусы, минуты, секунды | 55°45'21"N, 37°37'02"E | ⚠️ |

**Рекомендация:** Используйте десятичные градусы — это самый надёжный формат.

## 3.2. Построение маршрута

### 3.2.1. Создание простого маршрута

**Простой маршрут:** 2 точки (старт и финиш).

```
Шаг 1. Нажмите кнопку 🛤️ Построить маршрут
       Кнопка подсветится — режим активирован

Шаг 2. Кликните на карте в точке старта
       Появится синий маркер с буквой «A»

Шаг 3. Кликните на карте в точке финиша
       Появится красный маркер с буквой «B»
       Между точками проложится синяя линия

Шаг 4. Для завершения нажмите:
       - Кнопку 🔍 Анализ
       - Или клавишу Escape

Результат:
  ✅ Маршрут создан
  ✅ Дистанция рассчитана
  ✅ Можно выполнять анализ
```

### 3.2.2. Создание сложного маршрута

**Сложный маршрут:** 3 и более точек.

```
Шаг 1. Активируйте режим маршрута (🛤️)

Шаг 2. Последовательно кликайте на карте:
       Точка A (старт)
       Точка B (промежуточная)
       Точка C (промежуточная)
       Точка D (финиш)
       
       Каждая точка обозначается буквой
       Линии соединяют точки последовательно

Шаг 3. Для завершения:
       - Нажмите 🔍 Анализ
       - Или дважды кликните на последней точке
       - Или нажмите Escape

Совет: Для замкнутого маршрута (кольцо)
последнюю точку разместите там же, где первая.

Результат:
  ✅ Маршрут из N точек создан
  ✅ Сегменты рассчитаны
  ✅ Общая дистанция показана
```

### 3.2.3. Редактирование маршрута

**Перемещение точки:**

```
1. Наведите курсор на маркер точки
2. Когда курсор изменится (✋), зажмите левую кнопку
3. Перетащите точку в новое место
4. Отпустите кнопку мыши
5. Маршрут автоматически обновится
```

**Добавление точки на сегмент:**

```
1. Кликните правой кнопкой на линии маршрута
2. В контекстном меню выберите «Добавить точку»
3. Новая точка добавится в этом месте
4. При необходимости перетащите её
```

**Удаление точки:**

```
Способ 1:
1. Кликните правой кнопкой на маркере точки
2. Выберите «Удалить точку»

Способ 2:
1. Кликните на маркере точки (выделение)
2. Нажмите клавишу Delete

Способ 3:
1. Зажмите Ctrl и кликните на маркере
2. Точка удалится
```

**Изменение порядка точек:**

```
К сожалению, прямой перестановки точек нет.
Для изменения порядка:
1. Запишите координаты точек
2. Удалите маршрут (Escape)
3. Постройте заново в нужном порядке
```

# 4. РАСШИРЕННЫЕ ФУНКЦИИ

## 4.1. Коррекция по фактическим данным

### 4.1.1. Когда применять коррекцию

**Применяйте коррекцию, если:**

- ✅ Вы находитесь на месте планируемого полёта
- ✅ У вас есть фактические данные (датчики, метеостанция)
- ✅ Прогноз Open-Meteo отличается от фактической погоды
- ✅ Хотите повысить точность прогноза на 15-25%

# 6. РЕШЕНИЕ ПРОБЛЕМ

## 6.1. Частые проблемы

### Проблема 1: «Геолокация не работает»

**Симптомы:**
- Ошибка «Не удалось определить местоположение»
- Таймаут определения

**Решение:**

```
Способ 1: Разрешить в браузере
1. Нажмите 🔒 в адресной строке
2. Выберите «Настройки сайта»
3. Найдите «Геолокация»
4. Выберите «Разрешить»
5. Обновите страницу (F5)

Способ 2: Проверить системные настройки
Windows:
  Параметры → Конфиденциальность → Геолокация
  → Разрешить приложениям доступ

macOS:
  Системные настройки → Защита → Геолокация
  → Разрешить браузеру

Способ 3: Ввести координаты вручную
1. Используйте ручной ввод (раздел 3.1)
2. Найдите координаты в Google Maps
```

### Проблема 2: «Прогноз не загружается»

**Симптомы:**
- Ошибка «Failed to fetch»
- Бесконечная загрузка

**Решение:**

```
1. Проверьте интернет-соединение
   - Откройте любой сайт
   - Если не работает — восстановите соединение

2. Проверьте доступность API
   - Откройте: https://api.open-meteo.com/
   - Если не открывается — проблема на стороне API

3. Попробуйте другие координаты
   - Введите: 55.7558, 37.6173 (Москва)
   - Если работает — проблема с исходными координатами

4. Очистите кэш браузера
   - Ctrl+Shift+Delete
   - Выберите «Кэшированные изображения и файлы»
   - Нажмите «Удалить»

5. Попробуйте другой браузер
```

### Проблема 3: «Маршрут не сохраняется»

**Симптомы:**
- После перезагрузки маршрут исчезает
- Ошибка сохранения

**Решение:**

```
1. Проверьте режим браузера
   - В режиме инкогнито localStorage недоступен
   - Выйдите из режима инкогнито

2. Проверьте квоту хранилища
   - Откройте консоль (F12)
   - Введите: console.log(localStorage.length)
   - Если много данных — очистите старые

3. Разрешите cookies и localStorage
   Chrome:
     Настройки → Конфиденциальность → Cookies
     → Разрешить сохранение данных
   
   Firefox:
     Настройки → Приватность → Cookies
     → Разрешить сайтам сохранять данные

4. Экспортируйте маршрут в KML
   - Это резервная копия
   - Можно импортировать позже
```

### Проблема 4: «PDF не экспортируется»

**Симптомы:**
- Ошибка при экспорте
- Пустой файл

**Решение:**

```
1. Убедитесь, что анализ выполнен
   - Без данных экспорт невозможен
   - Нажмите 🔍 Анализ

2. Проверьте загрузку pdfmake
   - Откройте консоль (F12)
   - Введите: console.log(typeof pdfMake)
   - Должно быть: "object"
   - Если "undefined" — обновите страницу

3. Уменьшите состав отчёта
   - Отключите некоторые разделы
   - Попробуйте снова

4. Попробуйте другой браузер
   - Некоторые браузеры блокируют скачивание
```

## 6.2. Сообщения об ошибках

| Сообщение | Причина | Решение |
|-----------|---------|---------|
| «Неверные координаты» | Координаты вне диапазона | Широта: -90..90, долгота: -180..180 |
| «Превышено количество точек» | >50 точек в маршруте | Удалите лишние точки |
| «Ошибка сети» | Нет интернета | Проверьте подключение |
| «Недостаточно данных» | Не выполнен анализ | Нажмите «Анализ» |
| «LocalStorage недоступен» | Заблокирован браузером | Разрешите в настройках |
| «CORS error» | Запрос с локального файла | Используйте веб-сервер или GitHub Pages |
