# 1. ВВЕДЕНИЕ

## 1.1. Назначение документа

Настоящее руководство оператора (далее — РО) предназначено для пользователей системы MIRA v0.1.5.0 (далее — Система) и содержит сведения о подготовке к работе, рабочем режиме, аварийных ситуациях и правилах эксплуатации.

## 1.3. Краткое описание Системы

MIRA — веб-приложение для метеорологического обеспечения полётов беспилотных воздушных судов (БВС).

**Основные возможности:**
- 🌤️ Получение прогноза погоды от Open-Meteo
- 🗺️ Построение маршрутов по карте
- ⚠️ Анализ рисков вдоль маршрута
- 🚩 Коррекция по фактическим данным
- 📄 Экспорт отчётов в PDF

## 4.3. Построение маршрута

### 4.3.1. Создание маршрута

```
Шаг 1. Нажмите кнопку 🛤️ Маршрут
       (кнопка подсветится — режим активен)

Шаг 2. Кликните на карте — точка A (старт)
       Появится маркер с буквой «A»

Шаг 3. Кликните на карте — точка B (финиш)
       Появится маркер с буквой «B»
       Между точками проложится линия маршрута

Шаг 4. При необходимости добавьте промежуточные точки:
       Кликните — точка C, D, E...

Шаг 5. Для завершения нажмите:
       - Кнопку 🔍 Анализ (автоматически завершает)
       - Или кликните правой кнопкой на карте
```

### 4.3.2. Редактирование маршрута

**Перемещение точки:**
```
1. Наведите курсор на маркер точки

3. Перетащите точку в нужное место
4. Отпустите кнопку мыши
```

**Удаление точки:**
```
1. Кликните правой кнопкой на маркере точки
2. Выберите «Удалить точку»

ИЛИ:
1. Нажмите клавишу Delete
2. Кликните на маркере точки
```

**Добавление точки на сегмент:**
```
1. Кликните правой кнопкой на линии маршрута
2. Выберите «Добавить точку»
3. Новая точка добавится в этом месте

### 4.4.2. Анализ маршрута

```
Шаг 1. Постройте маршрут (раздел 4.3)
Шаг 2. Нажмите кнопку 🔍 Анализ
Шаг 3. Дождитесь обработки (5-10 секунд для 100 км)

Результат:
  ✅ Прогноз получен для всех сегментов
  ✅ Риски рассчитаны по сегментам
  ✅ Построены графики

### 4.4.3. Интерпретация результатов

**Статус полёта:**

| Статус | Значение | Действия |
|--------|----------|----------|
| 🟢 РАЗРЕШЁН | Risk Score 0-2 | Полёт разрешён |
| 🟡 ОГРАНИЧЕН | Risk Score 3-5 | Полёт возможен с ограничениями |
| 🔴 ЗАПРЕЩЁН | Risk Score 6-8 | Полёт не рекомендуется |

**Уровни риска параметров:**

| Параметр | LOW 🟢 | MEDIUM 🟡 | HIGH 🔴 |
|----------|--------|-----------|---------|
| Ветер 10м | ≤10 м/с | 10-15 м/с | >15 м/с |
| Ветер 500м | ≤15 м/с | 15-20 м/с | >20 м/с |
| Осадки | ≤1,4 мм/ч | 1,4-5 мм/ч | >5 мм/ч |
| Видимость | ≥5 км | 2-5 км | <2 км |

**Пример интерпретации:**
```
СТАТУС: 🟡 ОГРАНИЧЕН
Risk Score: 4/8

ПАРАМЕТРЫ:
  Ветер 10м:    12 м/с 🟡 (MEDIUM)
  Ветер 500м:   18 м/с 🟡 (MEDIUM)
  Осадки:       0 мм/ч 🟢 (LOW)
  Видимость:    10 км 🟢 (LOW)
  Обледенение:  LOW 🟢

РЕКОМЕНДАЦИИ:
  - Полёт возможен с ограничениями
  - Максимальная высота: 400 м
  - Следить за ветром на высотах >350 м
  - Иметь запасной вариант посадки
```

# 5. АВАРИЙНЫЕ СИТУАЦИИ

## 5.1. Типовые проблемы и решения

### Проблема 1: Приложение не загружается

**Симптомы:**
- Белый экран
- Бесконечная загрузка

**Решение:**
```
1. Проверьте подключение к интернету
2. Обновите страницу (F5 или Ctrl+R)
3. Очистите кэш браузера (Ctrl+Shift+Delete)
4. Попробуйте другой браузер
5. Проверьте доступность GitHub Pages
```

### Проблема 2: Не определяется геолокация

**Симптомы:**
- Ошибка «Геолокация недоступна»
- Таймаут определения

**Решение:**
```
1. Проверьте разрешение браузера:
   - Нажмите 🔒 в адресной строке
   - Разрешите геолокацию
2. Включите GPS (на мобильных устройствах)
3. Введите координаты вручную
4. Перезагрузите страницу
```

### Проблема 3: Не получается прогноз

**Симптомы:**
- Ошибка «Failed to fetch»
- Пустые данные

**Решение:**
```
1. Проверьте подключение к интернету
2. Подождите 1-2 минуты, попробуйте снова
3. Проверьте координаты (должны быть в диапазоне:
   широта -90..90, долгота -180..180)
4. Попробуйте другие координаты (например, Москва)
```

### Проблема 4: Не сохраняется маршрут

**Симптомы:**
- После перезагрузки маршрут исчезает

**Решение:**
```
1. Проверьте, что не в режиме инкогнито
2. Разрешите cookies и localStorage
3. Очистите старые данные:
   - Откройте консоль (F12)
   - Введите: localStorage.clear()
   - Перезагрузите страницу
4. Сохраните маршрут заново
```

### Проблема 5: Не экспортируется PDF

**Симптомы:**
- Ошибка при экспорте
- Пустой файл

**Решение:**
```
1. Убедитесь, что анализ выполнен
2. Проверьте наличие данных для отчёта
3. Попробуйте уменьшить количество разделов
4. Обновите страницу и повторите
```

## 5.2. Сообщения об ошибках

| Сообщение | Причина | Решение |
|-----------|---------|---------|
| «Неверные координаты» | Координаты вне диапазона | Проверьте формат ввода |
| «Превышено количество точек» | >50 точек в маршруте | Удалите лишние точки |
| «Ошибка сети» | Нет интернета | Проверьте подключение |
| «Недостаточно данных» | Не выполнен анализ | Нажмите «Анализ» |
| «LocalStorage недоступен» | Заблокирован браузером | Разрешите в настройках |

```

Шаг 3. Если данные есть, они восстановятся автоматически
Шаг 4. Если данных нет, загрузите из резервной копии
```

## Приложение Г. Часто задаваемые вопросы

**В: Как часто обновляется прогноз?**
О: Прогноз кэшируется на 1 час. Для обновления нажмите «Анализ» снова.

**В: Можно ли использовать MIRA без интернета?**
О: Нет, требуется подключение к Open-Meteo API для получения прогноза.

**В: Сколько маршрутов можно сохранить?**
О: Количество не ограничено, но есть ограничение по объёму localStorage (~5 МБ).

**В: Можно ли экспортировать маршрут в другой формат?**
О: Да, поддерживается экспорт в KML.

**В: Как поделиться маршрутом с коллегой?**
О: Экспортируйте в KML и отправьте файл, или дайте ссылку на GitHub Pages.

**В: Точны ли данные MIRA?**
О: MIRA использует данные Open-Meteo — это модель прогноза. Фактические данные могут отличаться. Используйте коррекцию по фактическим наблюдениям для повышения точности.
