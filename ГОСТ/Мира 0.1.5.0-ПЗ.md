```

**Алгоритм получения прогноза:**
```
1. Формирование URL API:
   https://api.open-meteo.com/v1/forecast?
     latitude={lat}&longitude={lon}&
     hourly={параметры}&
     daily={параметры}&
     current_weather=true&
     timezone=auto&
     forecast_days=7

2. Параметры запроса:
   - temperature_2m, relative_humidity_2m, dew_point_2m
   - precipitation, rain, showers, snowfall
   - cloud_cover, pressure_msl
   - wind_speed_10m, wind_direction_10m, wind_gusts_10m
   - visibility

3. Отправка запроса (fetch)
4. Обработка ответа (JSON)
5. Кэширование результата
6. Возврат данных
```

**Алгоритм расчёта Risk Score:**
```
Вход: hourly — почасовые данные, thresholds — пороги

Risk Score = 0

Для каждого часа:
  1. Ветер 10м:
     если windSpeed > 15 м/с → +2
     если windSpeed > 10 м/с → +1
  
  2. Ветер 500м:
     если windSpeed500 > 20 м/с → +2
     если windSpeed500 > 15 м/с → +1
  
  3. Осадки:
     если precipitation > 5 мм/ч → +2
     если precipitation > 1.4 мм/ч → +1
  
  4. Видимость:
     если visibility < 2 км → +2
     если visibility < 5 км → +1
  
  5. Обледенение:
     если icingRisk == HIGH → +2
     если icingRisk == MEDIUM → +1
  
  6. Турбулентность:
     если turbulenceRisk == HIGH → +2
     если turbulenceRisk == MEDIUM → +1

Уровень риска:
  Risk Score = 0-2 → LOW (зелёный)
  Risk Score = 3-5 → MEDIUM (жёлтый)
  Risk Score = 6-8 → HIGH (красный)

Статус полёта:
  LOW → РАЗРЕШЁН
  MEDIUM → ОГРАНИЧЕН
  HIGH → ЗАПРЕЩЁН
```

### 4.2.3. RouteModule (js/route.js)

**Назначение:** Управление маршрутами, сегментация, KML

**Основные функции:**

    calculateDistance(p1, p2) // Расчёт дистанции
    exportToKML(route)      // Экспорт в KML
    importFromKML(kmlText)  // Импорт из KML
}
```

**Алгоритм сегментации маршрута:**
```
Вход: points — массив точек [{lat, lon}, ...]

Сегменты = []
currentDistance = 0
segmentStart = points[0]

Для каждой точки i от 1 до N-1:
  dist = calculateDistance(points[i-1], points[i])
  currentDistance += dist
  
  Если currentDistance >= 10 км:
    // Завершение сегмента
    segment = {
      start: segmentStart,
      end: points[i],
      distance: currentDistance
    }
    Сегменты.push(segment)
    
    // Начало нового сегмента
    segmentStart = points[i]
    currentDistance = 0

// Добавление последнего сегмента
Если currentDistance > 0:
  Сегменты.push({start: segmentStart, end: points[N-1], distance: currentDistance})

Выход: Сегменты — массив сегментов по ~10 км
```

**Формула расчёта дистанции (Haversine):**
```javascript
function calculateDistance(lat1, lon1, lat2, lon2) {
    const R = 6371; // Радиус Земли, км
    const dLat = (lat2 - lat1) * Math.PI / 180;
    const dLon = (lon2 - lon1) * Math.PI / 180;
    const a = 
        Math.sin(dLat/2) * Math.sin(dLat/2) +
        Math.cos(lat1 * Math.PI / 180) * Math.cos(lat2 * Math.PI / 180) *
        Math.sin(dLon/2) * Math.sin(dLon/2);
    const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
    return R * c;
}
```

### 4.2.4. PilotModule (js/pilot.js)

**Назначение:** Коррекция прогноза по фактическим данным пилота

**Основные функции:**

```javascript
const PilotModule = {
    pilotData: null,         // Данные пилота
    
    init()                  // Инициализация
    setPilotData(data)      // Установка данных
    getCurrentLocation()    // Получение геолокации
    applyCorrection()       // Применение коррекции
    calculateBias()         // Расчёт коэффициентов
}
```

**Алгоритм коррекции прогноза:**
```
Вход:
  - forecast: прогноз Open-Meteo
  - pilotData: фактические данные

1. Расчёт коэффициентов:
   windBias = pilotData.windSpeed / forecast.windSpeed10m
   tempOffset = pilotData.temperature - forecast.temperature2m
   humidityBias = pilotData.humidity / forecast.humidity2m

2. Применение к высотам с затуханием:
   Для каждой высоты h (250, 350, 450, 550 м):
     weight = exp(-h / 1000)  // Затухание
     
     correctedWind[h] = forecast.wind[h] × windBias × weight + 
                        forecast.wind[h] × (1 - weight)
     
     correctedTemp[h] = forecast.temp[h] + tempOffset × weight

3. Коррекция видимости и облачности:
   Если pilotData.visibility < forecast.visibility:
     correctedVisibility = pilotData.visibility
   
   Если pilotData.cloudBase < forecast.cloudBase:
     correctedCloudBase = pilotData.cloudBase

4. Пересчёт рисков:
   recalculatedRiskScore = calculateRiskScore(correctedData)

5. Формирование рекомендаций:
   Если correctedWind[550м] > 15 м/с:
     maxAltitude = 400м
   Иначе:
     maxAltitude = 550м

Выход:
  - correctedForecast: скорректированный прогноз
  - recommendations: рекомендации
  - maxAltitude: максимальная высота
```

# 6. ПРИЛОЖЕНИЯ

## Приложение А. Формулы расчётов

### А.1. Расчёт точки росы (упрощённый)

```
Td = T - ((100 - RH) / 5)

где:
  Td — точка росы (°C)
  T — температура (°C)
  RH — относительная влажность (%)
```

### А.2. Расчёт риска обледенения

```
Условия обледенения:
  - Температура: от -10°C до +5°C
  - Влажность: > 80%
  - Видимость: < 5 км (туман)

Риск:
  Если все 3 условия → HIGH
  Если 2 условия → MEDIUM
  Если 0-1 условие → LOW
```

### А.3. Расчёт риска турбулентности

```
Условия турбулентности:
  - Ветер > 10 м/с
  - Порывы > 15 м/с
  - Разница порывы/ветер > 5 м/с

Риск:
  Если все 3 условия → HIGH
  Если 2 условия → MEDIUM
  Если 0-1 условие → LOW
```
