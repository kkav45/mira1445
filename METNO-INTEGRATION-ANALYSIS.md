## 🎯 Постановка задачи

### Текущая реальность:
1. Пилот недоволен прогнозом Open-Meteo
2. Вводит фактические данные в форму **«Сидя на земле»**
3. Проводится коррекция прогноза по фактическим данным
4. Результаты показываются во вкладке **«Расширенный анализ»** (шаг 3)

### Предлагаемое улучшение:
Добавить **третий источник данных** — **api.met.no** (Норвежский метеорологический институт) для сравнения:
- **Open-Meteo** (прогноз)
- **api.met.no** (альтернативный прогноз)
- **Фактические данные пилота** (наблюдения с земли)

   - Ссылка: https://api.met.no/

## 🏗️ Архитектура интеграции

### Уровень 1: Модуль MetNoModule

```javascript
/**
 * MIRA - MetNoModule (api.met.no)
 * Locationforecast 2.0 API
 */

const MetNoModule = {
    API_BASE: 'https://api.met.no/weatherapi/locationforecast/2.0',
    USER_AGENT: 'MIRA-Weather-App/0.1.5.0 (contact@example.com)',
    cachedData: {},

    /**
     * Получение прогноза для точки
     */
    async getForecast(lat, lon) {
        const cacheKey = `metno_${lat.toFixed(2)}_${lon.toFixed(2)}`;
        
        // Проверка кэша (2 часа)
        const cached = Storage.getFromCache(cacheKey);
        if (cached) return cached;

        const url = `${this.API_BASE}/compact?lat=${lat}&lon=${lon}`;
        
        const response = await fetch(url, {
            headers: {
                'User-Agent': this.USER_AGENT
            }
        });

        if (!response.ok) {
            throw new Error(`MET No API error: ${response.status}`);
        }

        const data = await response.json();
        Storage.saveToCache(cacheKey, data, 120); // 2 часа
        
        return data;
    },

    /**
     * Парсинг данных MET No в формат MIRA
     */
    parseForecast(metNoData) {
        const timeseries = metNoData.properties.timeseries;
        
        const hourly = timeseries.map(ts => ({
            time: ts.time,
            temp2m: ts.data.instant.details.air_temperature,
            humidity: ts.data.instant.details.relative_humidity_2m,
            wind10m: ts.data.instant.details.wind_speed,
            windDir: ts.data.instant.details.wind_from_direction,
            pressure: ts.data.instant.details.air_pressure_at_sea_level,
            cloudCover: ts.data.instant.details.cloud_area_fraction,
            precip: ts.data.next_1_hours?.details.precipitation_amount || 0,
            weatherCode: this.mapWeatherCode(ts.data.next_1_hours?.summary?.symbol_code)
        }));

        return {
            hourly: hourly,
            source: 'metno',
            generatedAt: metNoData.properties.meta.updated_at
        };
    },

    /**
     * Маппинг кодов погоды MET No → MIRA
     */
    mapWeatherCode(metNoCode) {
        const mapping = {
            'clear_sky': 0,
            'fair': 1,
            'partly_cloudy': 2,
            'cloudy': 3,
            'rain': 61,
            'heavy_rain': 63,
            'snow': 71,
            'thunder': 95
        };
        return mapping[metNoCode] || 0;
    }
};
```

## 🔢 Математика сравнения источников

### 1. Расчёт риска для каждого источника

```javascript
function calculateSourceRisk(sourceData) {
    let score = 0;
    
    // Ветер (вес 30%)
    if (sourceData.wind10m > 10) score += 3;
    else if (sourceData.wind10m > 7) score += 2;
    else if (sourceData.wind10m > 5) score += 1;
    
    // Видимость (вес 25%)
    if (sourceData.visibility < 3) score += 3;
    else if (sourceData.visibility < 5) score += 2;
    
    // Осадки (вес 20%)
    if (sourceData.precip > 1) score += 2;
    else if (sourceData.precip > 0.5) score += 1;
    
    // Температура/обледенение (вес 15%)
    if (sourceData.temp <= 0 && sourceData.humidity > 80) score += 2;
    
    // Облачность (вес 10%)
    if (sourceData.cloudCover > 80) score += 1;
    
    return score;
}
```

### 2. Доверие к источникам

```javascript
const confidence = {
    // Пилот — 100% доверия (фактические данные)
    pilot: 1.0,
    
    // Open-Meteo — экспоненциальное затухание
    openMeteo: (hoursFromNow) => Math.exp(-hoursFromNow / 24),
    
    // MET No — аналогично, но с коэффициентом
    metNo: (hoursFromNow) => 0.9 * Math.exp(-hoursFromNow / 24)
};
```

### 3. Расхождение между прогнозами

```javascript
function calculateDiscrepancy(openMeteo, metNo) {
    const windDiff = Math.abs(openMeteo.wind10m - metNo.wind10m);
    const tempDiff = Math.abs(openMeteo.temp2m - metNo.temp2m);
    const humidityDiff = Math.abs(openMeteo.humidity - metNo.humidity);
    
    const discrepancy = {
        wind: windDiff,
        temp: tempDiff,
        humidity: humidityDiff,
        total: (windDiff / 10) + (tempDiff / 10) + (humidityDiff / 100)
    };
    
    // Интерпретация
    if (discrepancy.total < 0.3) return 'Низкое расхождение ✅';
    if (discrepancy.total < 0.6) return 'Среднее расхождение ⚠️';
    return 'Высокое расхождение ❗';
}
```

## ⚠️ Потенциальные проблемы

### 1. Rate Limiting

**Проблема:** MET No может блокировать частые запросы.

**Решение:**
- Кэширование на 2 часа
- Задержка между запросами 200мс
- Лимит: 1 запрос в 5 минут на точку

### 2. Разные форматы данных

**Проблема:** Open-Meteo и MET No используют разные форматы.

**Решение:**
- Унификация через `WeatherModule.normalizeForecast()`
- Общий внутренний формат MIRA

### 3. Производительность

**Проблема:** 2 API запроса + коррекция = медленно.

**Решение:**
- Параллельные запросы (`Promise.all`)
- Прогресс-индикатор
- Фоновая загрузка
