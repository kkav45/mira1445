# 🚀 MIRA - Автоматическая загрузка на GitHub

## 📋 Быстрый старт

### 1️⃣ Первоначальная настройка (один раз)

**Запустите:**
```
init-auto-upload.bat
```

**Что делает:**
- ✅ Инициализирует Git репозиторий
- ✅ Настраивает пользователя (kkav45)
- ✅ Добавляет удалённый репозиторий
- ✅ Сохраняет учётные данные
- ✅ Выполняет первую загрузку

**При первом запуске:**
- Введите логин GitHub
- Введите пароль или Personal Access Token

---

### 2️⃣ Автоматическая загрузка (постоянно)

**Запустите:**
```
start-auto-deploy.bat
```

**Что делает:**
- 🟢 Мониторит изменения файлов (HTML, CSS, JS, и другие)
- 🟢 Автоматически создаёт коммит
- 🟢 Загружает на GitHub
- 🟢 Работает в фоновом режиме

**Остановить:** `Ctrl+C` в окне консоли

---

## 🔑 Personal Access Token

Если не принимает пароль:

1. Откройте: https://github.com/settings/tokens
2. **Generate new token (classic)**
3. Отметьте: `repo`, `workflow`, `write:packages`
4. **Generate token**
5. Скопируйте токен (показывается один раз!)
6. Используйте вместо пароля при загрузке

## ⚙️ Настройки

### Изменить отслеживаемые файлы

Откройте `auto-deploy.ps1`, измените строку:
```powershell
$filter = "*.*"  # Все файлы (кроме .git и tmp)
```

### Изменить частоту проверки

В `auto-deploy.ps1`:
```powershell
Start-Sleep -Seconds 1  # Задержка перед загрузкой
```

## ❓ Решение проблем

### "Git не найден"
→ Установите Git: https://git-scm.com/download/win

### "Repository not found"
→ Убедитесь, что репозиторий существует на GitHub

### "Authentication failed"
→ Используйте Personal Access Token вместо пароля

### "Permission denied"
→ Запустите от имени администратора
