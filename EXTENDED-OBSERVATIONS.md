# 🌤️ MIRA: Расширенная система метеонаблюдений

## Версия: 2026022703

---

## 🎯 Назначение

Система позволяет вносить **множественные фактические наблюдения** о погоде с различных координат и временных точек для повышения точности прогноза.

---

## 📊 Принцип работы

### 1. Сбор наблюдений

Пользователь вносит данные с указанием:
- **Координат** (широта, долгота)
- **Времени** наблюдения
- **Метеопараметров** (ветер, температура, влажность, видимость и др.)

```
Наблюдение 1: 10:00, Москва (55.75, 37.61), Ветер 5 м/с, Темп +2°C
Наблюдение 2: 11:00, Москва (55.75, 37.61), Ветер 7 м/с, Темп +3°C
Наблюдение 3: 12:00, Химки (55.89, 37.43), Ветер 9 м/с, Темп +2°C
...
```

### 2. Взвешенная коррекция

Для каждого часа прогноза система рассчитывает **весовые коэффициенты** на основе:

#### А) Временное затухание
```javascript
timeWeight = exp(-|t_observation - t_forecast| / 12 часов)
```

| Разница во времени | Вес |
|--------------------|-----|
| 0 часов (сейчас)   | 100% |
| 3 часа             | 78%  |
| 6 часов            | 61%  |
| 12 часов           | 37%  |
| 24 часа            | 14%  |

#### Б) Пространственное затухание (будущая версия)
```javascript
distanceWeight = exp(-distance_km / 50 км)
```

| Расстояние | Вес |
|------------|-----|
| 0 км       | 100% |
| 25 км      | 61%  |
| 50 км      | 37%  |
| 100 км     | 14%  |

### 3. Применение коррекции

Для каждого параметра рассчитывается **взвешенная поправка**:

```javascript
weightedBias = Σ((факт/прогноз - 1) * weight) / Σweight
коррекция = прогноз * (1 + weightedBias)
```

        weights = sorted.map(obs => ({
            obs,

        totalWeight = sum(weights);

## 📈 Сценарии использования

### Сценарий 1: Мониторинг с одной точки

**Ситуация**: Пилот находится на точке взлета и вносит данные каждые 30 минут.

```
10:00 → Ветер 5 м/с, Темп +2°C
10:30 → Ветер 6 м/с, Темп +2.5°C
11:00 → Ветер 7 м/с, Темп +3°C
11:30 → Ветер 6 м/с, Темп +3.5°C
```

**Результат**: Система усредняет наблюдения с учётом времени, повышая точность краткосрочного прогноза.

### Сценарий 2: Мониторинг с разных точек

**Ситуация**: Несколько пилотов в разных точках маршрута вносят данные.

```
Пилот 1 (Москва):     10:00, Ветер 5 м/с, Темп +2°C
Пилот 2 (Химки):      10:15, Ветер 7 м/с, Темп +1°C
Пилот 3 (Подольск):   10:30, Ветер 4 м/с, Темп +3°C
```

**Результат**: Система учитывает пространственные различия и корректирует прогноз для каждого сегмента маршрута.

### Сценарий 3: Долгосрочный мониторинг

**Ситуация**: Пилот вносит данные в течение нескольких дней.

```
День 1, 10:00: Ветер 5 м/с, Темп +2°C
День 2, 10:00: Ветер 8 м/с, Темп +0°C
День 3, 10:00: Ветер 3 м/с, Темп +5°C
```

**Результат**: Система накапливает статистику для улучшения долгосрочных прогнозов.

## 🎯 Преимущества новой системы

### Было (одно наблюдение):
- ❌ Только одна точка замера
- ❌ Только одно время
- ❌ Затухание 24 часа
- ❌ 3 корректируемых параметра

### Стало (множественные наблюдения):
- ✅ **Множество точек** (разные координаты)
- ✅ **Множество времён** (история наблюдений)
- ✅ **Гибкое затухание** (12 часов для каждого наблюдения)
- ✅ **5 параметров** (ветер, температура, влажность, видимость, явления)
- ✅ **Накопление данных** (история в localStorage)
- ✅ **Экспорт/импорт** (резервное копирование)
